# 三、推荐论文总结构

## 第 1 章 绪论

### 1.1 研究背景

写现代文生图模型在语义理解、图像质量、风格生成上进展显著，但对低频全局属性，尤其是亮度、曝光、背景纯度、背光剪影，仍可能存在控制不稳定。这里不要写得太泛，要直接落到你的任务：

> 文生图模型容易生成语义正确但亮度不符合 prompt 的图像，例如 dark scene 被生成成中等亮度，white background 出现灰底或复杂纹理，black background 不够纯净，backlight silhouette 前景不够暗。

SDXL 1.0 是 Stability AI 发布的开放模型，Diffusers 也提供 SDXL 相关 LoRA 训练支持，因此适合作为你的主要训练骨干；SD3.5 是 Stability AI 2024 年发布的开放模型系列，官方强调多个变体可定制、可在消费级硬件运行，因此适合纳入现代冻结基线；FLUX.1-dev 可作为另一类现代开放权重模型进行冻结推理对比。([Stability AI](https://stability.ai/news-updates/stable-diffusion-sdxl-1-announcement?utm_source=chatgpt.com))

### 1.2 研究问题

建议明确写成 5 个 RQ：

**RQ1：现有文生图模型在极端亮度和背景纯度控制上是否存在可测量缺口？**

**RQ2：BrightControlBench 是否能区分不同模型、不同训练策略和不同推理配置的低频控制能力？**

**RQ3：LoRA、offset noise、v_prediction、分段 α(t) 对亮度控制能力分别产生什么影响？**

**RQ4：亮度控制增益是否伴随语义一致性、自然度、多样性或中性场景表现下降？**

**RQ5：自动指标与 HumanCheck-100 人工盲评是否一致？哪些自动指标最能代表人类判断？**

### 1.3 研究内容与贡献

这一节按“基准—方法—实验—系统”四段写。注意不要把 offset noise 包装成完全原创算法，更适合写作：

> 本文不是提出一个大型新模型，而是围绕低频亮度控制问题，构建专用评测基准，并系统比较轻量微调与推理配置对控制效果的影响。

## 第 2 章 相关工作

### 2.1 文生图扩散模型

介绍 Stable Diffusion、SDXL、SD3.5、FLUX 等模型演进。注意这部分不要铺太多历史，重点服务你的模型选择：

- SDXL 1.0：主训练平台。
- SD3.5 Medium、FLUX.1-dev：现代冻结基线。

你的计划中已经把模型层次定义为“一主两辅一锚点”：SDXL 主训练，SD3.5 Medium 和 FLUX.1-dev 做冻结推理，SD1.5 作为 legacy anchor。

### 2.2 轻量微调与 LoRA

介绍 LoRA 的基本思想：冻结主体模型，只训练少量低秩适配参数。Diffusers 官方文档也说明 LoRA 是一种快速、轻量的训练方法，训练更少参数，通常权重文件更小，适合快速适配模型([Hugging Face](https://huggingface.co/docs/diffusers/api/loaders/lora?utm_source=chatgpt.com))。

本节结尾引出：你的任务不是个性化人物/风格，而是用 LoRA 研究亮度控制能力能否通过少量参数调整得到改善。

### 2.3 噪声预测、v-prediction 与 brightness-aware inference

介绍 epsilon prediction、v_prediction、zero terminal SNR、trailing timestep spacing、guidance rescale。Diffusers scheduler 文档明确说明，如果模型以 v_prediction 训练，可以启用 `rescale_betas_zero_snr=True` 和 `timestep_spacing="trailing"`，以支持极亮和极暗样本的生成([Hugging Face](https://huggingface.co/docs/diffusers/using-diffusers/schedulers?utm_source=chatgpt.com))。

你的论文里要非常谨慎：
**不要写“zero-SNR 对所有 checkpoint 都有效”。**
应写成：

> brightness-aware inference 是一个需要与训练目标匹配的推理配置，因此本文将其作为 v_prediction 相关组的推理变量，而不是直接套到所有 stock checkpoint 上。

这与你计划里“不要拿 stock 模型硬跑 zero-SNR 当主实验”的要求一致。

### 2.4 文生图评测基准

比较 TIFA、GenEval、T2I-CompBench、VQAScore/GenAI-Bench 与你的 BrightControlBench。核心论点：

- 现有 benchmark 很强，但主要测语义对齐、组合关系、对象属性、人类偏好。
- 你的 benchmark 专门测低频亮度、曝光、背景纯度。
- 因此你的工作不是重复已有 benchmark，而是补充一个更窄但更可测的控制维度。

### 2.5 人工评测与一致性分析

介绍人工盲评、评分者一致性、自动指标相关性。这里为第 3 章 HumanCheck-100 铺垫。

# 第 3 章 BrightControlBench 基准构建

这一章应作为论文核心之一，建议写得非常完整。

## 3.1 设计目标

写 5 个原则：

1. **低频属性专用性**：重点测亮度、曝光、背景纯度，而非一般语义对齐。
2. **类别覆盖性**：覆盖暗、亮、背光、纯白、纯黑、夜景、中性、复杂语义联合控制。
3. **可量化性**：每类 prompt 都能对应亮度均值、极端覆盖率、背景纯度等指标。
4. **可复现性**：固定 prompt、seed、resolution、sampler、steps、CFG。
5. **人类可校准性**：HumanCheck-100 用于验证自动指标是否符合人类判断。

## 3.2 PromptBench-160

建议写成 8 类 × 20 条：

| 类别                          | 目标                      | 典型 prompt 设计                           |
| ----------------------------- | ------------------------- | ------------------------------------------ |
| Extreme Dark                  | 生成极暗但语义可辨图像    | dark room, dim candle, low-key photography |
| Extreme Bright                | 高亮、雪地、强曝光        | snowy field, overexposed studio            |
| Night/Candle/Neon             | 局部光源与整体暗环境      | neon street, candle-lit dinner             |
| Backlight/Silhouette          | 前景暗、背景亮            | person standing against sunset             |
| White Background              | 纯白背景、商品摄影        | object on pure white background            |
| Black Background              | 纯黑背景、低调摄影        | object on pure black background            |
| Neutral Control               | 常规曝光对照组            | normally lit photo                         |
| Complex Semantic + Brightness | 多对象、属性、关系 + 亮度 | two objects under backlight                |

这一节要说明 prompt 不是随便写的，而是参考已有 text-to-image benchmark 的模板化组织方式，再替换为亮度控制目标。你计划里已经明确要求 PromptBench-160 分 8 类，每类 20 条，并建议至少一半 prompt 借鉴已有 benchmark 的模板范式。

## 3.3 DataBench-500

建议写 COCO val2017 中选 500 张：

- dark 200
- bright 200
- neutral 100

每张图取 COCO caption，加 brightness tag，形成 reference-based 测试集。COCO 官方定位为大规模 object detection、segmentation、captioning 数据集，适合作为公开、可复现的数据来源([COCO Dataset](https://cocodataset.org/?utm_source=chatgpt.com))。你的计划也已经明确用 COCO train2017 构建亮度平衡训练集，用 COCO val2017 构建 DataBench。

这里要解释为什么 DataBench-500 和 PromptBench-160 都需要：

- PromptBench-160：测试开放 prompt 下的指令控制能力。
- DataBench-500：测试有参考亮度分布时的客观误差。
- 两者互补，避免 benchmark 只依赖主观 prompt。

## 3.4 HumanCheck-100

建议写成严格盲评协议：

- 从最终模型结果中抽 100 个样本。
- 打乱模型名称与样本顺序。
- 每张图由 3 名评分者独立打分。
- 评分维度：
  - Brightness correctness：1–5
  - Prompt faithfulness：1–5
  - Background purity：1–5，仅对白底/黑底适用
  - Overall preference：可选，1–5

统计方法：

- 对 ordinal 评分：Krippendorff’s alpha 或 ICC。
- 对 pairwise 排名：Kendall’s W。
- 对二分类成功/失败：Fleiss’ kappa。
- 自动指标与人评：Spearman ρ，必要时补 Kendall τ。
- 给出 95% bootstrap confidence interval。

这样写能显著增强 benchmark 的可信度。你的计划已经明确要求完善 HumanCheck-100 盲评结果统计、计算评分者一致性、补充盲评结果与自动指标的相关性分析。

## 3.5 评测指标

建议定义为三组。

**亮度控制指标：**

- Luminance MAE
- Extreme Coverage Rate
- Direction Success Rate
- Histogram Wasserstein Distance

**背景纯度指标：**

- White Border Purity
- Black Border Purity
- Background Mask Purity，可选

**语义与质量约束指标：**

- VQAScore 或 TIFA
- CLIPScore，可作为辅助但不要作为唯一语义指标
- LPIPS / CLIP embedding dispersion，用于多样性
- HumanCheck 相关性

这里要强调：
**亮度指标不能单独代表模型好坏。**
一个模型可以通过把所有图都变暗来降低 dark MAE，但这不代表它真的理解了 brightness prompt。你当前 v2 结果正好说明这一点：v_prediction 的 dark MAE 最低，但报告指出这是因为它“均匀过暗化”，而不是选择性控制；同时 bright 子集上 offset005_ckpt2500 才是明确胜者。

# 第 4 章 方法与实验设置

这一章不要写成“我发明了一个全新模型”，而要写成“轻量微调变量设计”。

## 4.1 总体框架

输入 prompt，经文生图模型生成图像，然后用 BrightControlBench 指标评估。训练分为：

- frozen inference baseline
- LoRA baseline
- offset noise LoRA
- v_prediction LoRA
- offset noise + v_prediction
- 分段 α(t) offset noise

## 4.2 模型设置

建议写：

| 角色                    | 模型         | 用途                         |
| ----------------------- | ------------ | ---------------------------- |
| Legacy anchor           | SD1.5        | 保留已有实验，作为旧模型参照 |
| Main trainable backbone | SDXL 1.0     | 主训练消融                   |
| Modern frozen baseline  | SD3.5 Medium | 现代模型横评                 |
| Modern frozen baseline  | FLUX.1-dev   | 现代模型横评                 |

这样和你的计划完全一致：主训练骨干是 SDXL 1.0，现代冻结基线是 SD3.5 Medium、FLUX.1-dev，SD1.5 是 legacy anchor；训练变量包括 LoRA、offset noise、v_prediction、α(t)，推理变量包括 sampler、step 数和 brightness-aware scheduler。

## 4.3 LoRA 微调设置

写清楚：

- LoRA rank
- learning rate
- batch size / gradient accumulation
- resolution
- max steps
- checkpoint interval
- seed
- optimizer
- mixed precision
- 是否训练 text encoder
- 是否开启 xFormers / gradient checkpointing

你已有 v3 实验中记录了非常规范的训练设置：noise_offset=0.05 + v_prediction、6000 images、LoRA rank=8、max steps=1500、每 300 steps 存 checkpoint、seed=42、DDIM 30 steps、CFG 7.5。后续 SDXL 实验也应按这个粒度记录。

## 4.4 Offset noise 与 α 设计

建议写三层：

- 固定 α：0.02 / 0.05 / 0.08
- 选择最佳固定 α
- 分段 α(t)：根据 timestep 区间调节噪声偏移强度

注意：分段 α(t) 不一定要作为最终最好方法。它可以是“进一步探索 timestep 对低频控制的影响”。如果结果不好，也能作为负消融写入论文。

## 4.5 v-prediction 与 brightness-aware inference

这一节要非常科学地写：

- v_prediction 是官方 brightness control 路线中的训练目标之一。
- zero-SNR / trailing / guidance_rescale 是与 v_prediction 配套的推理配置。
- 但你的现有实验显示，v_prediction 不一定提升所有亮度控制任务，因此本文将其作为独立变量和组合变量进行系统比较。

你已有 v3 结果显示：offset005 + v_prediction 没有超过 offset005_ckpt2500，best vpred+offset 的 DataBench MAE 为 0.1292，高于 offset005_ckpt2500 的 0.1112；但它能降低 neutral damage，best neutral MAE 为 0.0671，低于 offset005_ckpt2500 的 0.1082。这说明你最终论文里不要简单写“二者互补显著优于单独方法”，而应写成：

> offset noise 是方向控制的主要增益来源；v_prediction 在组合设置中可能缓解中性场景过暗化，但会削弱方向成功率，体现出亮度控制强度与自然亮度保持之间的权衡。

# 第 5 章 实验设计

模型：

- SD1.5
- SDXL 1.0

数据：

目的：

2. 证明亮度控制问题不是 SD1.5 的历史问题。
3. 分析现代模型是否仍在极暗、极亮、纯背景、背光场景上存在 tradeoff。

## 5.2 实验二：SDXL 主训练消融

训练组建议保持你计划中的 5 组：

| 组别 | 方法                                     | 目的                           |
| ---- | ---------------------------------------- | ------------------------------ |
| A1   | LoRA baseline                            | 标准微调参照                   |
| B1   | LoRA + v_prediction                      | 官方 brightness-aware 训练参照 |
| C1   | LoRA + fixed offset noise                | 验证 offset noise 主效应       |
| C2   | LoRA + fixed offset noise + v_prediction | 验证二者交互                   |

你的计划中已经明确了这个 SDXL 主训练矩阵，并建议固定 α 先测 0.02 / 0.05 / 0.08，再基于最佳 α 做 α(t)。

## 5.3 实验三：checkpoint 演化分析

不要只报告 final checkpoint。你现在 v1/v2/v3 都有 checkpoint 分析，这是优势。v1 报告指出 ckpt2000 亮度抑制最强，但 ckpt2500 具有更好的 per-prompt differentiation，并且 bright prompt 更亮，因此选 ckpt2500 作为 best checkpoint。

论文中建议画：

- step vs Luminance MAE
- step vs Direction Success Rate
- step vs Bright MAE / Dark MAE / Neutral MAE
- step vs Border Purity
- step vs VQAScore

这能体现训练过程，而不是只展示最终数字。

## 5.4 实验四：推理鲁棒性

变量：

- sampler：DDIM / DPM-Solver / Euler
- steps：20 / 30 / 50
- CFG：例如 5.0 / 7.5 / 10.0
- brightness-aware inference：zero-SNR + trailing + guidance_rescale

目的：

- 判断训练得到的亮度控制是否依赖某个 sampler。
- 判断更多 steps 是否真的提高亮度控制。
- 判断 CFG 增大是否强化 prompt 遵循但破坏自然度。
- 分析推理配置对 gain-cost tradeoff 的影响。

这对应你计划中的 inference robustness：对最佳 2–3 个模型比较 sampler、step 数、guidance 配置，并加入 brightness-aware inference 设置。

## 5.5 实验五：HumanCheck-100 人工一致性与相关性

要单独成一个实验，而不是附录。

建议输出：

1. 三个评分维度的平均分和置信区间。
2. 评分者一致性。
3. 自动指标与人工评分相关性矩阵。
4. 自动综合分与人类 overall preference 的相关性。
5. 典型一致案例和不一致案例。

关键结论可以写成：

> 如果 Luminance MAE 与 brightness correctness 显著负相关，Border Purity 与 background purity 显著正相关，VQAScore/TIFA 与 prompt faithfulness 显著正相关，则说明 BrightControlBench 的自动评测具有较好人类一致性。

## 5.6 实验六：小规模跨模型泛化，可选冲优

如果时间够，做 SD3.5 Medium 的小规模 LoRA pilot：

- baseline LoRA
- best offset-noise 配置

数据只用 5k–10k 极端子集。这个实验不必成为主线，但可以放在“泛化性分析”中，增强论文层次。

## 6.4 Offset noise 的主效应

分析：

## 6.5 v-prediction 与 offset noise 的交互

这一节非常重要。你要写得诚实。

根据当前 v3，offset005 + v_prediction 应作为 **negative / mixed ablation**，而不是主方法。v3 报告明确建议该方法作为负/探索性消融，因为它没有超过 offset005_ckpt2500，且 v_prediction + noise_offset 可能引入冲突训练信号，说明 noise_offset alone 才是显式亮度控制的主要驱动。

建议论文措辞：

> 与预期不同，v_prediction 与 offset noise 的简单组合没有带来稳定互补。该组合在部分 checkpoint 上缓解 neutral damage，但显著降低方向成功率，说明低频亮度控制不是单调增强问题，而需要在极端控制与自然曝光保持之间平衡。

这比强行说“互补效果显著”更科学。

## 6.8 Gain-cost tradeoff 总结

这是结果章的结尾，也是答辩最容易讲清楚的地方。

建议总结成三类方法：

| 类型     | 代表                      | 优点                     | 代价                   | 论文结论         |
| -------- | ------------------------- | ------------------------ | ---------------------- | ---------------- |
| 保守型   | baseline LoRA / offset002 | 语义和中性场景稳定       | 极端亮度控制弱         | 适合自然图像     |
| 强控制型 | offset005                 | 方向成功率高，极端控制强 | neutral over-darkening | 适合亮度控制任务 |
| 混合型   | offset005 + v_prediction  | 可能保护中性亮度         | 方向控制不稳定         | 作为负/交互消融  |

# 第 8 章 结论与展望

## 8.1 研究总结

按 RQ 回答：

- BrightControlBench 能否揭示低频控制缺口？
- 哪些模型/方法表现最好？
- offset noise 的主效应是什么？
- v_prediction 的作用是否符合预期？
- 自动指标和人评是否一致？
- gain-cost tradeoff 是什么？

## 8.2 局限性

必须写，且会让论文更可信：

- BrightControlBench 主要关注亮度和背景纯度，不覆盖所有视觉属性。
- HumanCheck-100 样本量有限。
- SD3.5 / FLUX 主要作为冻结基线，没有大规模训练。
- 自动亮度指标可能受图像内容分布影响。
- prompt 语言、风格、摄影术语会影响模型生成。

## 8.3 未来工作

- 扩大 benchmark 到更多低频属性：色温、对比度、雾化、阴影。
- 引入区域级亮度控制。
- 使用更强的 VLM 做自动评估。
- 构建公开 leaderboard 或网页测评平台。
- 探索更稳定的 timestep-aware noise schedule。
